Warum CORS? Frontend (Port 8080) und Backend (Port 8081) laufen als separate Prozesse. Browser blockieren standardmäßig Cross-Origin-Anfragen. WebConfig.java erlaubt Anfragen von localhost:* explizit via @Configuration + CorsRegistry.

2. Ein Spielzug — Schritt für Schritt

Schritt 0 / 6
1
Spieler drückt ENTER
Der Tastendruck landet im globalen keydown-Listener in app.js. Er ruft handleInput("Enter") auf, was wiederum submitGuess() auslöst.
// app.js — globaler Tastatur-Listener document.addEventListener("keydown", (event) => { if (event.metaKey || event.ctrlKey || event.altKey) return; if (document.activeElement === elements.guessInput) { if (event.key === "Enter") { event.preventDefault(); submitGuess(); // ← hier geht's weiter } return; } handleInput(event.key); });
2
fetch() sendet HTTP POST
submitGuess() baut einen JSON-Body und sendet ihn per fetch() an das Backend. Die Funktion ist async — der Browser wartet auf die Antwort ohne die Seite zu blockieren.
// app.js — submitGuess() async function submitGuess() { setBusy(true); try { state.game = await requestJson( `/games/${state.game.gameId}/guesses`, { method: "POST", body: JSON.stringify({ guess: state.currentGuess }) } ); // → POST http://localhost:8081/api/games/{uuid}/guesses // → Body: { "guess": "STEIN" } } finally { setBusy(false); render(); } }
3
GameController empfängt den Request
Spring Boot deserialisiert den JSON-Body automatisch in ein GuessRequest-Record. @PathVariable liest die UUID aus der URL. Der Controller delegiert sofort an den Service.
// GameController.java @PostMapping("/{gameId}/guesses") public GameStateResponse submitGuess( @PathVariable UUID gameId, @RequestBody GuessRequest request) { // GuessRequest = record { String guess; } return gameService.submitGuess(gameId, request); }
4
GameService bewertet den Rateversuch
evaluateGuess() vergleicht Buchstaben in zwei Durchläufen: zuerst alle exakten Treffer (CORRECT), dann die Buchstaben im falschen Feld (PRESENT), der Rest bleibt ABSENT.
// GameService.java — evaluateGuess() GuessEvaluation evaluateGuess(String guess, String solution) { // 1. Alle Buchstaben zunächst ABSENT List<LetterEvaluation> letters = new ArrayList<>(guess.length()); for (int i = 0; i < guess.length(); i++) letters.add(new LetterEvaluation(String.valueOf(guess.charAt(i)), LetterState.ABSENT)); // 2. Erster Durchlauf: CORRECT markieren Map<Character, Integer> remaining = new HashMap<>(); for (int i = 0; i < solution.length(); i++) { if (solution.charAt(i) == guess.charAt(i)) letters.set(i, new LetterEvaluation(String.valueOf(guess.charAt(i)), LetterState.CORRECT)); else remaining.merge(solution.charAt(i), 1, Integer::sum); } // 3. Zweiter Durchlauf: PRESENT (Buchstabe falsch platziert) for (int i = 0; i < guess.length(); i++) { if (letters.get(i).state() == LetterState.CORRECT) continue; int rem = remaining.getOrDefault(guess.charAt(i), 0); if (rem > 0) { letters.set(i, new LetterEvaluation(String.valueOf(guess.charAt(i)), LetterState.PRESENT)); remaining.put(guess.charAt(i), rem - 1); } } return new GuessEvaluation(guess, List.copyOf(letters)); }
5
GameStateResponse wird als JSON zurückgesendet
Spring Boot serialisiert das Java-Record automatisch zu JSON. solution ist nur enthalten, wenn das Spiel bereits beendet ist — sonst null, damit das Lösungswort verborgen bleibt.
// GameStateResponse.java — Record (automatisch JSON-serialisierbar) public record GameStateResponse( UUID gameId, int wordLength, int maxAttempts, int attemptCount, GameStatus status, // IN_PROGRESS | WON | LOST List<GuessEvaluation> guesses, String message, String solution // null solange IN_PROGRESS ) {} // Beispiel-JSON-Antwort: // { // "gameId": "a1b2-...", // "status": "IN_PROGRESS", // "attemptCount": 1, // "guesses": [{ "word": "STEIN", // "letters": [ // {"letter":"S","state":"ABSENT"}, // {"letter":"T","state":"CORRECT"}, // {"letter":"E","state":"PRESENT"}, // {"letter":"I","state":"ABSENT"}, // {"letter":"N","state":"ABSENT"} // ]}], // "solution": null // }
6
renderBoard() malt die farbigen Kacheln
Der Browser empfängt die JSON-Antwort. state.game wird aktualisiert, dann ruft render() alle Darstellungsfunktionen auf. renderBoard() erstellt DOM-Elemente für jede Kachel mit CSS-Klassen tile--correct, tile--present oder tile--absent.
// app.js — renderBoard() (vereinfacht) function renderBoard() { elements.board.innerHTML = ""; for (let row = 0; row < MAX_ATTEMPTS; row++) { const pastGuess = state.game?.guesses?.[row]; for (let col = 0; col < wordLength; col++) { if (pastGuess) { const ev = pastGuess.letters[col]; // ev.state = "CORRECT" | "PRESENT" | "ABSENT" row.append(createTile(ev.letter, ev.state, /*revealed=*/true)); } } } } // createTile setzt die CSS-Klassen: // tile--correct → grün (richtiger Buchstabe, richtige Position) // tile--present → gelb (Buchstabe im Wort, falsche Position) // tile--absent → grau (Buchstabe nicht im Wort) // Visuelles Ergebnis für "STEIN" (Lösung: "TINTE"): const demo = [ { letter: "S", bg: "#3a3a3c" }, // ABSENT { letter: "T", bg: "#b59f3b" }, // PRESENT ~ { letter: "E", bg: "#b59f3b" }, // PRESENT ~ { letter: "I", bg: "#b59f3b" }, // PRESENT ~ { letter: "N", bg: "#b59f3b" }, // PRESENT ~ ];
