Warum CORS? Frontend (Port 8080) und Backend (Port 8081) laufen als separate Prozesse. Browser blockieren standardmäßig Cross-Origin-Anfragen. WebConfig.java erlaubt Anfragen von localhost:* explizit via @Configuration + CorsRegistry.
